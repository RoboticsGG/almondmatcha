BASE STATION - GROUND CONTROL
=============================
Location: Ground station PC (Windows/Linux/WSL)
Domain: Default (telemetry only)

SETUP
-----
Source ROS2 environment:
$ . ~/ros2_humble/install/local_setup.bash

Optional: Source in ~/.bashrc for auto-load:
$ echo ". ~/ros2_humble/install/local_setup.bash" >> ~/.bashrc

BUILD
-----
Build mission control package:
$ cd ~/almondmatcha/ws_base
$ colcon build
$ source install/setup.bash

LAUNCH
------
Launch mission control and monitoring:
$ export ROS_DOMAIN_ID=2
$ cd ~/almondmatcha/ws_base
$ source install/setup.bash
$ ros2 run mission_control node_commands --ros-args \
  --params-file ./src/mission_control/config/params.yaml

Or with launcher:
$ ros2 launch mission_control node_comlaunch.py

MONITORING
----------
Monitor mission status (in separate terminal):
$ ros2 run mission_control node_monitoring

Check active nodes:
$ ros2 node list

Monitor GNSS mission data:
$ ros2 topic echo /tpc_gnss_mission_active
$ ros2 topic echo /tpc_gnss_mission_remain_dist
$ ros2 topic echo /tpc_gnss_spresense

Monitor rover control commands:
$ ros2 topic echo /pub_rovercontrol_d2

CONFIGURATION
--------------
Edit mission parameters:
$ nano ~/almondmatcha/ws_base/src/mission_control/config/params.yaml

Key parameters:
- des_lat: Destination latitude
- des_long: Destination longitude
- speed_limit: Maximum rover speed
- timeout: Command timeout (ms)

TROUBLESHOOTING
---------------
Check ROS2 environment:
$ printenv | grep ROS

Verify network connectivity to rover:
$ ping 192.168.1.1
$ ping 192.168.1.5

Check topic visibility:
$ ros2 topic list

Enable debug logging:
$ ros2 run mission_control node_commands --log-level debug
